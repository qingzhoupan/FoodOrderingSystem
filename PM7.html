Documentation: Architecture:
  1. The welcome page. We have a start.html page, it’s the welcome page for our ordering system. In the welcome page our users can choose “Eat in” and the “Take Out”.
  2. The Navigation menu. In our system we have the navigation menu which includes 6 different buttons represent 6 different categories.
  3. The Payment page. And we also have a payment page in our system. we ask user to choose one of the payment methods. They need to choose “debit” or “credit” in this page.
  4. The scroll bar. The main page is scrollable when a category has more than 4 items.
  5. The search box. We also have a search box which located top of the main page, it allowed user type keywords and it will show results for that kind of food.
  6. The Pop-up message. If users items cannot found, a popup message will shows.

Documentation:Differences from Design:
  1. We delete the “Back” button in the main page. The first reason is we have technical difficulties. The second reason is we keep the navigation buttons and the search bar. Therefore, the “Back” button is useless for users.
  2. We added the pop-up message in our system. When a customer using search box type something doesn’t exist, the pop-up message shows.
  3. In our paper prototype, we want to change the “My Order” button. When people added something the “My Order” button shows the item already exist in “My Order List”, like shopping cart.
Project Conclusion and Reflection

    Change

  1. About the text color which inside the button, we don’t change the its color, we just enlarged the font to make it’s more clear for users.
  2. We need to add a introduction for the users who use our system first.

    What features are necessary to add to current prototype to test for readiness.

  1. We can add a “help” button which provides staff service at the corner of the interface.
  2. We can add a membership system to allow users to have their own accounts. And they can earn points with each purchase. Customers can redeem points for gifts or dishes.

    Key points

  1. At the beginning of the design, we should keep font size and font color same in one row or column. This is helpful when we try to keep the consistency.
  2. Do not make things small, it is hard for users and you to read.
  3. Categorizing things can decrease the amount of work. Also, interface can look more clear and in alignment.
  4. A rigorous and orderly html is good for typesetting for the interface and also for css file when we want to find the certain div.
  5. At css file, some divs have the same feature, we can put all selectors at on property. It not only can save space, but also make us easy to find the property when we want to change later.
  6. For the class and id, we need to give some meaningful name. That’s because when we need to change some properties of certain div, a meaningful name will help us find that div quickly and correctly.
  7. The font size is always a big problem. We even don’t have a standard that how large font we need to use.
